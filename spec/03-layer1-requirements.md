# Layer 1 - Requirements Layer

## Purpose

Define what a valid ingredient substitute must satisfy. Given an ingredient and optional context (product category, region), generate a structured set of quality requirements based on industry standards (USP, FCC, FDA, EU regulations, Codex Alimentarius).

## Status: Implemented

Located at `src/requirement_layer/`.

## Architecture

```
Input JSON
  |
  v
InputProcessor          -- validate + normalize input
  |
  v
RequirementEngine       -- Gemini agent + Google Search Grounding
  |
  v
RuleValidator           -- validate rule logic, normalize units
  |
  v
IdGenerator             -- assign stable REQ-{INGREDIENT}-{SEQ} IDs
  |
  v
OutputFormatter         -- serialize to JSON output contract
  |
  v
Output JSON
```

## Input Contract

```json
{
  "schema_version": "1.0",
  "ingredient": {
    "ingredient_id": "ING-ASCORBIC-ACID",
    "canonical_name": "Ascorbic Acid",
    "aliases": ["Vitamin C"]
  },
  "context": {
    "product_category": "beverage",
    "region": "EU"
  },
  "baseline_supplier": null
}
```

## Output Contract

```json
{
  "schema_version": "1.0",
  "ingredient_id": "ING-ASCORBIC-ACID",
  "requirements": [
    {
      "requirement_id": "REQ-ING-ASCORBIC-ACID-001",
      "field_name": "assay_percent",
      "rule_type": "range",
      "operator": "between",
      "min_value": 99.0,
      "max_value": 100.5,
      "unit": "%",
      "priority": "hard",
      "source_reference": "USP"
    }
  ],
  "notes": "Generated by Layer 1 RequirementEngine for Ascorbic Acid"
}
```

## Components

### InputProcessor (`input_processor.py`)

- Validates input against `Layer1Input` Pydantic model
- Normalizes ingredient aliases (dedup, strip whitespace)
- Applies default context (`region: "global"`) when omitted
- Normalizes units via `UNIT_ALIASES` mapping:
  - `"mg/kg"` -> `"ppm"`
  - `"percent"` / `"percentage"` -> `"%"`
  - `"µm"` / `"micrometre"` -> `"um"`
  - `"°c"` / `"deg c"` -> `"celsius"`

### RequirementEngine (`requirement_engine.py`)

- Core LLM agent powered by Google Gemini via `google-genai` SDK
- Uses Google Search Grounding to find industry standards data
- Constructs a user prompt from ingredient name, region, product category, and aliases
- Sends a single Gemini API call with system instruction and Google Search tools
- Parses the JSON response (handles markdown fences, `requirements` key wrapping)
- Validates each rule via `RuleValidator` and assigns IDs via `IdGenerator`
- Implements exponential backoff retry on 429 rate limits (configurable `max_retries`, `retry_delay`)

**Configuration:**
- Default model: `gemini-3-flash-preview` (overridable via CLI `--model`)
- Requires: `GEMINI_API_KEY` environment variable
- Constants defined in `model_config.py`: `DEFAULT_MODEL`, `DEFAULT_MAX_RETRIES`, `DEFAULT_RETRY_DELAY`, `GEMINI_TOOLS`

### RuleValidator (`rule_validator.py`)

- Validates raw rule dicts against `RequirementRule` Pydantic model
- Normalizes units using `InputProcessor.normalize_unit()`
- Infers default operators from `rule_type` when omitted:
  - `range` -> `"between"`
  - `minimum` -> `">="`
  - `maximum` -> `"<="`
  - `enum_match` -> `"in"`
  - `boolean_required` -> `"=="`
  - `free_text_reference` -> `"reference"`
- Delegates field-specific validation to Pydantic `model_validator` on `RequirementRule`

### IdGenerator (`id_generator.py`)

- Generates stable sequential IDs: `REQ-{ingredient_id}-{sequence_number}`
- Maintains an internal counter per instance
- Sequence numbers are zero-padded: `REQ-ING-ASCORBIC-ACID-001`

### OutputFormatter (`output_formatter.py`)

- Builds `Layer1Output` Pydantic model from ingredient ID, requirements list, and optional notes
- Serializes to dict (excludes None fields) or JSON string
- Writes to file with configurable indentation

## Pydantic Models (`schemas/models.py`)

### Enums
- `Priority`: `hard`, `soft`
- `RuleType`: `range`, `minimum`, `maximum`, `enum_match`, `boolean_required`, `free_text_reference`

### Input Models
- `IngredientRef`: `ingredient_id`, `canonical_name`, `aliases[]`
- `IngredientContext`: `product_category?`, `region?`
- `Layer1Input`: `schema_version`, `ingredient`, `context?`, `baseline_supplier?`

### Output Models
- `RequirementRule`: All fields from the shared contract, plus Pydantic `model_validator` enforcing rule-type-specific field constraints
- `Layer1Output`: `schema_version`, `ingredient_id`, `requirements[]`, `notes?`
- `ErrorResponse`: `schema_version`, `error`, `detail?`

## Responsibility Boundaries

**Responsible for:**
- Benchmark mapping from industry standards
- Hard vs soft requirement classification
- Requirement schema definition and validation
- Unit normalization

**Not responsible for:**
- Finding suppliers
- Retrieving supplier evidence
- Final recommendation
